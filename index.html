Tyrant Unleashed Optimizer

Optimze Your Deck
Usage Install FAQ
Tyrant Unleashed Optimizer is a command line tool to optimize your tyrant unleashed deck. It is in alpha test at the moment. So please do not distribute the link yourself. To start the command line interpreter in windows use Start: cmd.exe and navigate to the folder containing tu_optimizer.exe.

Simulate your deck against another deck

>tu_optimize "Cyrus, Medic, Revolver, Imperial APC" "Malika, Grunt, Blitz Armor" sim 100000
Your Deck: [RWFDFhH3] Cyrus, Medic, Revolver, Imperial APC
Enemy's Deck: [PyA0Fl] Malika, Grunt, Blitz Armor
win%: 59.443 (59443 / 100000)
stall%: 0.014 (14 / 100000)
loss%: 40.543 (40543 / 100000)
  • cards are identified by their name. By definition Cyrus means Cyrus Level 4. If you want to use Cyrus Level 1 use Cyrus-1
  • if you have eg. three copies of Revolver use Revolver(3)
  • by default your deck and enemy's deck are played in random order

Play your deck ordered

>tu_optimize "Cyrus, Imperial APC, Revolver, Medic" "Malika, Grunt, Blitz Armor" -r sim 100000
Your Deck: [RWFDH3Fh] Cyrus, Imperial APC, Revolver, Medic
Enemy's Deck: [PyA0Fl] Malika, Grunt, Blitz Armor
win%: 86.192 (86192 / 100000)
stall%: 0.088 (88 / 100000)
loss%: 13.72 (13720 / 100000)
  • -r tries to play your card in the correct order still following the "three card hand limit" rule
  • by default normal PVP mode is used. You play your first card in turn 1

Play your deck in surge mode

>tu_optimize "Cyrus, Imperial APC, Revolver, Medic" "Malika, Grunt, Blitz Armor" -r -s sim 100000
Your Deck: [RWFDH3Fh] Cyrus, Imperial APC, Revolver, Medic
Enemy's Deck: [PyA0Fl] Malika, Grunt, Blitz Armor
win%: 64.898 (64898 / 100000)
stall%: 0.083 (83 / 100000)

  • -s means enemy's deck starts in turn 1. You play your first card in turn 2

Use build in help if you forget command syntax

>tu_optimize
Tyrant Unleashed Optimizer 0.8
usage: tu_optimize Your_Deck Enemy_Deck [Flags] [Operations]

Your_Deck:
  the name/hash/cards of a custom deck.

Enemy_Deck:
  semicolon separated list of defense decks, syntax:
  deck1[:factor1];deck2[:factor2];...
  where deck is the name/hash/cards of a mission, raid, quest or custom deck, and factor is optional. The default factor is 1.
  example: 'eerie-spam:0.2;nbd-spam:0.8' means eerie-spam is the defense deck 20% of the time, while nbd-spam is the defense deck 80% of the time.

Flags:
  -r: the attack deck is played in order instead of randomly (respects the 3 cards drawn limit).
  -s: use surge (default is fight).
  -t : set the number of threads, default is 4.
  -turnlimit : set the number of turns in a battle, default is 50 (can be used for speedy achievements).
  win:     simulate/optimize for win rate. [default].
  defense: simulate/optimize for win rate + stall rate. can be used for defending deck.
Flags for climb:
  -c: don't try to optimize the commander.
  -L  : restrict deck size between  and .
  -o: restrict to the owned cards listed in "data/ownedcards.txt".
  -o=: restrict to the owned cards listed in .
  target : stop as soon as the score reaches .

Operations:
  sim : simulate  battles to evaluate a deck.
  climb : perform hill-climbing starting from the given attack deck, using up to  battles to evaluate a deck.
  reorder : optimize the order for given attack deck, using up to  battles to evaluate an order.
  debug: testing purpose only. very verbose output. only one battle.
  debuguntil  : testing purpose only. fight until the last fight results in range [, ]. recommend to redirect output.
  example: debuguntil 100 100 will run till first win.
  example: debuguntil 0 0 will run till first lose.
